 2. WEB SCRAPING & HTTP
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

 • requests  (v2.34.2)
   - Sends HTTP POST to the government portal API endpoint
     (https://agriwelfare.gov.in/en/getMOMDetail) to fetch PDF metadata.
   - Sends HTTP GET requests with browser-like User-Agent headers to download
     binary PDF files from the portal.
   - Used in: downloader.py, agri_website.py

 • BeautifulSoup4  (v4.15.0)
   - Parses the HTML "Details" field returned in the API JSON response.
   - Strips HTML tags to extract clean plain-text descriptions for each report.
   - Used in: agri_website.py

   Parser backend:
 • html.parser  (Python standard library)
   - Built-in HTML parser used by BeautifulSoup to process the Details field.

 3. PDF PROCESSING
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

 • PyMuPDF  (fitz)  (v1.26.0)
   - PRIMARY PDF processing engine.
   - Text Extraction   : page.get_text("text") — extracts Unicode text per page.
   - Block Layout      : page.get_text("blocks") — extracts text with bounding
                         box coordinates (x0, y0, x1, y1) per paragraph block.
   - Image Extraction  : page.get_images(full=True) + fitz.Pixmap() — decodes
                         embedded images from PDF XRef streams.
   - Image Positioning : page.get_image_rects(xref) — gives the spatial bbox of
                         an image within the page coordinate system.
   - Color Normalisation: fitz.Pixmap(fitz.csRGB, pix) converts CMYK/grayscale
                          to RGB; pix.alpha removal composites onto white.
   - Page Metadata     : page.rect gives page width and height.
   - Used in: extractors.py (TextExtractor, ImageExtractor, MetadataExtractor)

 • pdfplumber  (v0.11.5)
   - SECONDARY PDF engine, specialized for table detection.
   - Uses computer-vision line-intersection analysis and whitespace clustering
     to detect table boundaries within a PDF page.
   - page.find_tables() + table_obj.extract() → list-of-lists (rows × columns).
   - Provides table bounding boxes (bbox).
   - Used in: extractors.py (TableExtractor)

 • Pillow  (PIL)  (v11.2.1)
   - Image processing library used alongside PyMuPDF.
   - Handles image format conversion and safe JPEG saving for images that
     PyMuPDF cannot save directly (e.g., alpha-bearing or palette images).
   - Used in: extractors.py (ImageExtractor)

 4. DATA HANDLING & MANIPULATION
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

 • pandas  (v3.0.3)
   - Converts extracted table data (list-of-lists) into DataFrames.
   - Serialises tables to CSV (df.to_csv) and JSON (df.to_json, orient="records").
   - Date parsing and filtering: pd.to_datetime() with dayfirst=True, used to
     filter PDF records within a user-specified date range.
   - Used in: extractors.py (TableExtractor), downloader.py, agri_website.py

 • NumPy  (>=1.23, <2.0)
   - Provides the float32 numpy array infrastructure for embedding vectors.
   - Used for stacking embedding arrays into 2-D matrices for FAISS input.
   - Cosine similarity computed as dot product of L2-normalised vectors:
       sim = np.dot(emb_i, emb_{i+1})
   - Used in: hierarchical_chunker.py, embeddings.py, faiss_index.py, search.py

 • json  (Python standard library)
   - Serialises and deserialises all intermediate pipeline artifacts:
     metadata JSON, merged document JSON, chunk JSON, embedding JSON,
     FAISS metadata JSON.
   - Used in: all pipeline modules

 • pathlib.Path  (Python standard library)
   - All file path construction and directory creation uses pathlib for
     cross-platform compatibility and clean path operations.
   - Used in: all pipeline modules

 • logging  (Python standard library)
   - Structured logging across all pipeline phases via setup_logger() in utils.py.
   - Log files written to: data/logs/pipeline.log

 5. MACHINE LEARNING & EMBEDDINGS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

 • sentence-transformers  (v3.3.1)
   - HIGH-LEVEL wrapper around Hugging Face Transformers for sentence embeddings.
   - Model used: sentence-transformers/all-MiniLM-L6-v2
       Architecture : MiniLM (6-layer distilled BERT)
       Output dims  : 384 dimensions per text input
       Normalisation: L2-normalised (normalize_embeddings=True), enabling
                      cosine similarity via simple dot product.
   - Two roles in the pipeline:
       (a) Semantic Child Chunking — encodes text elements during splitting to
           detect natural topic boundaries via cosine similarity valleys.
       (b) Embedding Indexing — encodes all final child chunks for FAISS storage.
   - Model is cached in-process (function attribute) to avoid reloading cost.
   - Used in: hierarchical_chunker.py, embeddings.py, search.py

 • transformers  (v4.46.1)   [Hugging Face]
   - BACKEND framework underlying sentence-transformers.
   - Provides the tokenizer, attention mechanism, and transformer model weights.
   - Not called directly in the pipeline; used internally by sentence-transformers.

 • torch  (PyTorch)  (v2.5.1)
   - DEEP LEARNING FRAMEWORK underlying both transformers and sentence-transformers.
   - Runs model inference (forward passes) during encoding.
   - Not called directly; used internally by sentence-transformers for
     GPU/CPU tensor computation.

 6. VECTOR SEARCH & INDEXING
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

 • faiss-cpu  (v1.8.0.post1)   [Facebook AI Similarity Search]
   - Stores all 384-dim child chunk embedding vectors in a flat binary index.
   - Index type: IndexFlatL2
       • Exact brute-force L2 (Euclidean) distance search.
       • Every query vector is compared against all stored vectors.
       • Guarantees exact nearest-neighbour results (no approximation).
   - index.add(float32 matrix) — bulk loads all chunk vectors.
   - index.search(query_vector, top_k) — returns top_k closest chunk indices
     and their L2 distances in one vectorised call.
   - Index serialised to disk: data/faiss/chunks.index (binary format).
   - Metadata (chunk dicts) stored alongside: data/faiss/chunks_metadata.json
   - Used in: faiss_index.py, search.py
