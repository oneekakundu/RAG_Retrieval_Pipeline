Tech Stack for Crop Weather Watch RAG Pipeline
Programming Language

Python 3.11 (used throughout the project)
Core Libraries / Frameworks

PyMuPDF (fitz) – PDF parsing, page layout extraction, image extraction.
sentence‑transformers – Semantic text embeddings for chunk splitting and search.
FAISS – Vector similarity search and indexing of embedded chunks.
NumPy / SciPy – Numerical operations supporting embeddings and FAISS.
json – Standard library for storing intermediate representations (documents, chunks, metadata).
pathlib – Path handling across Windows.
RAG Pipeline Specific Packages

rag_pipeline package containing modules:
extractors.py – Image and table extraction.
chunking.py – Page‑level chunk creation, linking of images/tables.
hierarchical_chunker.py – Parent‑child chunk hierarchy, semantic splitting.
linking.py – Establish previous/next links and layout ordering.
search.py – Search interface returning text plus linked images/tables.
pipeline.py – Orchestrates end‑to‑end processing, embedding, index building.
run_rag_pipeline.py – CLI entry point.
Embedding & Indexing

sentence‑transformers model (e.g., all-MiniLM-L6-v2) for generating dense vectors.
FAISS index (IndexFlatIP or similar) for fast similarity lookup.
Data Storage

JSON files for:
Raw extracted text (data/text/...)
Tables (data/tables/... as CSV/JSON)
Images (data/images/... as JPEG)
Chunk metadata (data/chunks/...)
Embeddings (data/embeddings/...)
Pandas (optional) for CSV handling of tables.
CLI / Execution

run_rag_pipeline.py – command‑line interface using argparse for date range selection.
Standard library argparse, logging for configuration and progress reporting.
Testing

pytest – Unit tests located in tests/ covering chunking, linking, extraction, and search.
Development & Environment

Windows OS (tested on Windows 10/11).
Virtual environment located at d:\Webscraping\venv.
requirements.txt (or similar) lists the above dependencies.
Additional Utilities

git – Version control (repository tracking).
pip – Dependency installation within the virtual environment.
json and os for file system interactions.
This document captures all major components and libraries used across the project.
